# Contributing

Thanks for helping improve this project.

## Setup

- Use a recent Node version (v22 recommended).
- Run `npm test` before opening a PR.

## Pull Requests

- Keep PRs focused and small.
- Include a short test plan in the PR body.
- Prefer clear commit messages that explain intent.

## Coding Guidelines

- Keep UI logic in `js/ui.js` and core rules in `js/chess-engine.js`.
- Avoid introducing external runtime dependencies unless necessary.
- Preserve existing local-play behavior.
